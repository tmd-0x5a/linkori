Linkori Backend: Function Map

画像処理 & 検証

commands/image.rs
read_image_as_data_url

画像をBase64形式で取得する。ブラウザで即座に表示したい単体画像用。

高速化read_image_thumbnail

シークバー等のホバー時に使う縮小画像を生成。ヘッダーのみ読み取りで高速化。

read_directory_thumbnail

フォルダやZIPの表紙画像（1枚目）を取得する。

validate_image_file

画像の存在・形式・縦横サイズをチェックする。

チャンク (範囲選択) ロジック

commands/chunk.rs
核心validate_chunk

「AからBまで」という範囲が正常かチェックし、中身のリストを返す。Linkoriの心臓部。

resolve_chunk_images

ビューア起動時に、指定された範囲に含まれる全画像パスを確定させる。

配信プロトコル

protocol/manga.rs
高速化handle_manga_request

manga:// スキームのリクエストを処理。Base64の33%肥大化を避け、バイナリを直接流す。

image_path_to_manga_url

通常のローカルパスを、ブラウザが表示できる manga:// URL に変換する。

ZIP解析

reader/zip.rs
高速化load_meta

ZIPの中身（Central Directory）を解析。LRUキャッシュにより2回目以降は一瞬で終わる。

read_image_from_zip

ZIPを解凍せず、メモリ上で対象のエントリ（画像）だけをピンポイントで読み出す。

ファイル操作

commands/filesystem/*.rs
browse_directory / browse_zip

フォルダやZIPの中身をエクスプローラーのように一覧表示する（ブラウジング用）。

get_path_meta

ファイルの更新日時等を取得。プレイリストのソートに使用。

list_split_candidates

「サブフォルダで分割」機能の際、分割対象となる候補をリストアップする。

文字コード & ソート

utils/*.rs
decode_zip_path (encoding.rs)

ZIP内の文字化け（Shift-JIS/CP437）を自動で正常な日本語に直す。

natural_compare (sorting.rs)

「1.jpg, 2.jpg, 10.jpg」の順で並ぶようにする自然順ソートロジック。

find_zip_boundary (paths.rs)

パスのどこまでがZIPファイル名で、どこからが内部のフォルダなのかを切り分ける。
