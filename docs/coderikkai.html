01 — Overview

Linkori とは何か

ローカルにある画像フォルダ・ZIP / CBZ・PDF を、チャンクと呼ぶ連続区間に切り分けてまとめ、シームレスに読める Windows 向けデスクトップアプリです。

📂 フォルダ / ZIP の分割候補

1 つのフォルダをドラッグしたチャンクを、右クリック →「サブフォルダで分割」で、子フォルダと子 ZIP / CBZ を 1 チャンクずつに割ってくれる。巻ごとに ZIP で固めている書庫を一発でプレイリスト化できる。

🖼️ フォルダ / ZIP のサムネ自動生成

ファイルブラウザーでフォルダや ZIP をグリッド表示したとき、自然順で最初に見つかった画像を縮小してカードに表示。中身が想像できるので迷わない。

🚫 「要分割」バッジと親切エラー

フォルダ直下に画像がなく ZIP や子フォルダしかない場合、赤い NG ではなく青い「要分割」バッジに切り替わり、ツールチップで分解を促す。

🎯 read_directory_thumbnail の追加

フォルダ / ZIP どちらでも、先頭 1 枚の縮小 JPEG を返す新コマンド。フロントの useFileThumbnail が内部で呼び分ける。

03 — Types

型定義 — データの骨格

どのコンポーネントも、下の 5 つの型さえ押さえれば全部読めます。実ファイルは src/types/index.ts。

Chunk と Playlist

Chunk（連続区間）は startPath と endPath の 2 点だけで区間を表します。endPath が空文字なら「startPath 丸ごと」という意味になります（フォルダ全体・ZIP 全体・PDF 全体）。 

// src/types/index.ts
export interface Chunk {
  id: string;
  name?: string;              // 表示名（空なら #1 のようにインデックス表示）
  startPath: string;          // フォルダ / ZIP / 画像ファイル のパス
  endPath: string;            // 空なら start 丸ごと
}

export interface Playlist {
  id: string;
  name: string;
  chunks: Chunk[];
  createdAt: number;
  updatedAt: number;
  isFavorite?: boolean;
  tags?: string[];
  accent?: AccentKey;           // 背表紙色（タグ色パレット）
}

FileEntry — ブラウザで 1 行に対応する

browse_directory と browse_zip がこの配列を返します。is_dir / is_zip / is_image / is_pdf で「何のアイコンを出すか」を UI 側が決めます。 

export interface FileEntry {
  name: string;
  path: string;
  is_dir: boolean;
  is_zip: boolean;
  is_image: boolean;
  is_pdf: boolean;
  thumbnail_url: string | null;   // サーバー側からは null、フロントで取得して代入
  is_hidden: boolean;
  modified_at: number | null;     // Unix 秒
}

04 — State

状態管理 — Zustand ストア

状態は Redux 相当、ただしボイラープレート無しで書ける Zustand を使っています。ストアは 3 つだけ。

playlistStore

プレイリスト一覧 / アクティブ ID / チャンク CRUD / お気に入り / タグ。

ここが唯一の「真実の源」で、ChunkCard も PlaylistPanel もこの store しか触らない。

viewerStore

現在見ている画像パス配列、ページ番号、見開きや RTL の設定。

ビューアに入るときに resolveChunkImages の結果をここに流し込む。

settingsStore

アプリ全体設定（テーマ、言語、既定のビューア挙動）。

永続化対象なので usePersistence hook がディスクに同期する。

playlistStore の抜粋

// src/stores/playlistStore.ts（抜粋）
import { create } from "zustand";
import type { Chunk, Playlist } from "@/types";

interface PlaylistState {
  playlists: Playlist[];
  activePlaylistId: string | null;

  createPlaylist: (name: string) => Playlist;
  addChunk: (playlistId: string, startPath: string, endPath: string) => void;
  removeChunk: (playlistId: string, chunkId: string) => void;
  insertChunksAt: (playlistId: string, index: number, chunks: Chunk[]) => void;
  // 他 CRUD / 並び替え / お気に入り ...
}

export const usePlaylistStore = create<PlaylistState>()(( set, get ) => ({
  playlists: [],
  activePlaylistId: null,

  createPlaylist: (name) => {
    const pl: Playlist = {
      id: crypto.randomUUID(),
      name,
      chunks: [],
      createdAt: Date.now(),
      updatedAt: Date.now(),
    };
    set((s) => ({ playlists: [...s.playlists, pl] }));
    return pl;
  },

  insertChunksAt: (playlistId, index, newChunks) => {
    set((s) => ({
      playlists: s.playlists.map((p) =>
        p.id !== playlistId
          ? p
          : { ...p, chunks: [...p.chunks.slice(0, index), ...newChunks, ...p.chunks.slice(index)] }
      ),
    }));
  },
  // ...
}));
⚠️

永続化は store 内ではやらない。usePersistence hook が store の変化を購読して tauri-plugin-store に書き込みます。これでストアは純粋なメモリ操作に専念でき、テストもしやすい構造になっています。

05 — Bridge

Tauri 連携 — invoke 1 行の世界

Rust の関数を呼ぶときは、常に同じ形です。invoke("コマンド名", { 引数 })。戻り値は Promise。JS から見れば、ただの async 関数にしか見えません。

薄いラッパー src/lib/tauri.ts

全ての invoke はここを通します。コンポーネント側は型付きの関数名だけ呼べばよい。Rust コマンド名が変わってもここ 1 カ所で追従できる。 

// src/lib/tauri.ts（v0.6.0 抜粋）
import { invoke } from "@tauri-apps/api/core";
import type { FileEntry, ChunkValidationResult } from "@/types";

export async function validateChunk(
  startPath: string, endPath: string
): Promise<ChunkValidationResult> {
  return invoke<ChunkValidationResult>("validate_chunk", { startPath, endPath });
}

export async function browseDirectory(path: string): Promise<FileEntry[]> {
  return invoke<FileEntry[]>("browse_directory", { path });
}

// v0.6.0 で追加された 2 つ
export async function listSplitCandidates(
  path: string
): Promise<Array<{ name: string; path: string; kind: "folder" | "zip" }>> {
  return invoke("list_split_candidates", { path });
}

export async function readDirectoryThumbnail(
  path: string, maxSize: number = 120
): Promise<string> {
  return invoke("read_directory_thumbnail", { path, maxSize });
}
🧭

命名規約： JS 側は camelCase、Rust 側は snake_case。
Tauri が自動で変換してくれるので、maxSize は Rust 引数 max_size に、startPath は start_path に、何もしなくて繋がります。

画像は invoke じゃなく manga:// で

数百枚の画像バイト列を JS に渡すのは重い。だから画像だけは <img src="manga://...">で Rust から WebView に直送します。JS は URL を組み立てるだけ。 

// src/lib/mangaUrl.ts（考え方だけ抜粋）
export function imagePathToMangaUrl(path: string): string {
  // ZIP 内エントリの場合： zip://D:/foo.zip///chapter/001.jpg
  // 通常ファイルの場合： D:/foo/001.jpg
  // どちらも base64 化してクエリに乗せる
  return `manga://localhost/${kind}/${b64(path)}`;
}
06 — Rust Backend

Rust バックエンド — 仕事の取り仕切り

ここから Rust コードが出てきます。身構えなくて大丈夫。使う構文は 8 個だけです。

fn 関数名
関数宣言。引数と戻り値の型を書く。fn add(a: i32) -> i32 { ... }
async fn
await できる関数。Tauri コマンドは async にしておくと安全。
#[tauri::command]
「この関数を JS から invoke で呼べるようにする」印。
Result<T, E>
成功なら Ok(T)、失敗なら Err(E)。関数の戻り値でエラーを運ぶ。
?
Result を「成功なら取り出し、失敗なら早期 return」するショートカット。
match
強化版 switch。列挙型を全パターン捌く。
Option<T>
あるなし。Some(値) か None か。
&str / String
文字列スライス（読み取り用）と、所有権付き文字列（書き換え可）。

chunk.rs — 検証ロジックの要

渡されたパスを 4 種類の PathType に分類してから検証します。enum は列挙型。match で全部のパターンを漏れなく書くと、ビルド時に「漏れたパターンあるよ」と Rust が教えてくれます（型安全）。 

// src-tauri/src/commands/chunk.rs（要点だけ）
enum PathType {
    Directory(String),
    File(String, String),            // (ディレクトリ, ファイル名)
    ZipWhole(String),                 // ZIP ファイル全体
    ZipEntry(String, String),        // (ZIP パス, 内部エントリ)
}

fn classify_path(path: &str) -> PathType {
    let normalized = path.replace('\\', "/");
    if let Some((zip, inner)) = find_zip_boundary(&normalized) {
        match inner {
            Some(e) => PathType::ZipEntry(zip, e),
            None     => PathType::ZipWhole(zip),
        }
    } else if Path::new(&normalized).is_dir() {
        PathType::Directory(normalized)
    } else {
        /* File 分岐 ... */
    }
}

// v0.6.0: フォルダ直下に ZIP / サブフォルダがあるか？
fn has_splittable_children(dir: &Path) -> bool {
    let Ok(rd) = fs::read_dir(dir) else { return false; };
    for entry in rd.flatten() {
        let p = entry.path();
        if p.is_dir() { return true; }
        if p.is_file() && is_zip_like(&p) { return true; }
    }
    false
}

const SPLIT_HINT_MSG: &str =
    "フォルダ直下に画像がありません。ZIP/CBZ やサブフォルダを含むため、\
     右クリックメニューから『サブフォルダで分割』してください";

fn validate_dir_pair(dir1: &str, dir2: &str) -> ChunkValidationResult {
    if dir1 != dir2 { return err_result("異なるディレクトリ"); }
    let p = Path::new(dir1);
    match directory::list_images_recursively(p) {
        Ok(images) if !images.is_empty() => ok_result(/* paths */),
        Ok(_) => {
            if has_splittable_children(p) { err_result(SPLIT_HINT_MSG) }
            else { err_result("画像ファイルが見つかりません") }
        }
        Err(e) => err_result(e),
    }
}

フォルダ対応 — v0.6.0 の目玉実装

① list_split_candidates を ZIP 対応に拡張

実ファイル： src-tauri/src/commands/filesystem/meta.rs
フォルダ直下のサブフォルダと.zip / .cbzを自然順で 1 つの配列に混ぜて返します。UI が「フォルダアイコン」か「ZIP アイコン」か出し分けるために kind フィールドを付けます。 

// src-tauri/src/commands/filesystem/mod.rs
#[derive(Debug, Serialize)]
pub struct SplitCandidate {
    pub name: String,
    pub path: String,
    /// "folder" | "zip" — UI 側のアイコン切替用
    pub kind: &'static str,
}
// src-tauri/src/commands/filesystem/meta.rs（v0.6.0 新版）
#[tauri::command]
pub async fn list_split_candidates(path: String) -> Result<Vec<SplitCandidate>, String> {
    let normalized = path.replace('\\', "/");
    let dir = std::path::Path::new(&normalized);
    if !dir.is_dir() {
        return Err(format!("ディレクトリが存在しません: {path}"));
    }

    let mut entries: Vec<(std::path::PathBuf, &'static str)> = fs::read_dir(dir)
        .map_err(|e| format!("読み取り失敗: {e}"))?
        .filter_map(|e| e.ok())
        .map(|e| e.path())
        .filter_map(|p| {
            let name = p.file_name().and_then(|n| n.to_str()).unwrap_or("");
            if name.is_empty() || detect_is_hidden(&p, name) { return None; }
            if p.is_dir()           { Some((p, "folder")) }
            else if p.is_file() && is_zip_like(&p) { Some((p, "zip")) }
            else { None }
        })
        .collect();

    // 自然順（001, 002, ..., 10 ではなく 001, 002, 010 の並び）
    entries.sort_by(|a, b| {
        let na = a.0.file_name().and_then(|n| n.to_str()).unwrap_or("");
        let nb = b.0.file_name().and_then(|n| n.to_str()).unwrap_or("");
        natural_compare(na, nb)
    });

    Ok(entries.into_iter().map(|(p, kind)| SplitCandidate {
        name: p.file_name().and_then(|n| n.to_str()).unwrap_or("").to_string(),
        path: p.to_string_lossy().replace('\\', "/"),
        kind,
    }).collect())
}

② read_directory_thumbnail — フォルダ / ZIP のサムネ

実ファイル： src-tauri/src/commands/image.rs
フォルダなら直下の画像、ZIP なら内部の画像を自然順で先頭 1 枚取り、長辺 120px の JPEG（quality=25）に縮めて data URL で返します。速度優先。 

// src-tauri/src/commands/image.rs（v0.6.0 で追加）
#[tauri::command]
pub async fn read_directory_thumbnail(path: String, max_size: u32) -> Result<String, String> {
    let normalized = path.replace('\\', "/");
    let p = Path::new(&normalized);

    let (bytes, mime_path) = if p.is_dir() {
        let images = directory::list_images_in_directory(p)?;
        let Some(first) = images.into_iter().next() else {
            return Ok(String::new());       // 画像 0 枚なら空文字
        };
        (directory::read_image_from_directory(&first)?, first.to_string_lossy().to_string())
    } else if p.is_file() && is_zip_like(p) {
        let entries = zip_reader::list_images_in_zip(p)?;
        let Some(first) = entries.into_iter().next() else {
            return Ok(String::new());
        };
        (zip_reader::read_image_from_zip(p, &first)?, first)
    } else {
        return Err("対象はフォルダまたは ZIP/CBZ ファイルである必要があります".to_string());
    };

    // imagesize でヘッダだけ読んで寸法を得る（全デコード前に判定する軽量版）
    let threshold = max_size * 3;
    if let Ok(size) = imagesize::blob_size(&bytes) {
        if (size.width.max(size.height) as u32) <= threshold {
            // すでに十分小さい → そのまま data URL に
            let mime = mime_guess::from_path(&mime_path).first_or_octet_stream().to_string();
            let b64 = base64::engine::general_purpose::STANDARD.encode(&bytes);
            return Ok(format!("data:{mime};base64,{b64}"));
        }
    }

    // 大きい画像はデコード → 長辺 max_size に縮小 → JPEG q25 エンコード
    let img = image::load_from_memory(&bytes).map_err(|e| format!("画像デコードエラー: {e}"))?;
    let (w, h) = (img.width(), img.height());
    let scale = (max_size as f32 / w.max(h) as f32).min(1.0);
    let thumb = img.resize_exact(
        ((w as f32 * scale) as u32).max(1),
        ((h as f32 * scale) as u32).max(1),
        image::imageops::FilterType::Nearest,
    );
    let mut out: Vec<u8> = Vec::new();
    image::codecs::jpeg::JpegEncoder::new_with_quality(Cursor::new(&mut out), 25)
        .encode_image(&thumb)
        .map_err(|e| format!("サムネイル生成エラー: {e}"))?;
    let b64 = base64::engine::general_purpose::STANDARD.encode(&out);
    Ok(format!("data:image/jpeg;base64,{b64}"))
}

③ フロント側の useFileThumbnail を拡張

実ファイル： src/hooks/useFileThumbnail.ts
画像 / フォルダ / ZIP の 3 通りを同じ呼び方で扱える hook にしました。FileEntry の flag を見て、内部で呼ぶ Rust コマンドを切り替えます。 

// src/hooks/useFileThumbnail.ts（v0.6.0）
interface ThumbTarget {
  path: string; is_image: boolean; is_dir: boolean; is_zip: boolean;
}

export function useFileThumbnail(entry: ThumbTarget): string | null {
  const [src, setSrc] = useState<string | null>(null);

  useEffect(() => {
    let cancelled = false;
    const loader = entry.is_image
      ? readImageAsDataUrl(entry.path)
      : (entry.is_dir || entry.is_zip)
        ? readDirectoryThumbnail(entry.path, 120)
        : null;
    if (!loader) return;
    loader.then((url) => { if (!cancelled && url) setSrc(url); }).catch(() => {});
    return () => { cancelled = true; };
  }, [entry.path, entry.is_image, entry.is_dir, entry.is_zip]);

  return src;
}

⑤ 分解の実処理 — handleSplitChunk

const handleSplitChunk = useCallback(async (chunkId: string) => {
  const idx = playlist.chunks.findIndex((c) => c.id === chunkId);
  const original = playlist.chunks[idx];
  const candidates = await listSplitCandidates(original.startPath);
  if (candidates.length === 0) return;

  // folder / zip を問わず、各候補を 1 チャンクに変換
  const newChunks: Chunk[] = candidates.map((c) => ({
    id: crypto.randomUUID(),
    name: c.name,
    startPath: c.path,
    endPath: "",                              // 空 = 丸ごと
  }));

  removeChunk(playlist.id, chunkId);
  insertChunksAt(playlist.id, idx, newChunks);
  splitUndo.recordSplit({ originalChunk: original, addedIds: newChunks.map((c) => c.id) });
}, [playlist]);
